## Dasar Teori

Modul 15 membahas tiga aspek utama keamanan sistem operasi Linux, yaitu **Access Control (Permission)**, **Integritas (Hashing)**, dan **Konfidensialitas (Enkripsi)**.

### 15.1 Access Control (Permission)

Linux adalah sistem operasi multi-pengguna yang menerapkan konsep kepemilikan file dan permission berbasis Unix. Setiap file dimiliki oleh satu pengguna dan satu grup, dengan izin akses tersendiri. Terdapat dua jenis pengguna: pengguna sistem (untuk proses non-interaktif) dan pengguna biasa (untuk login interaktif).

Permission file dapat dilihat dengan perintah `ls -l` yang menampilkan mode dalam format 10 karakter. Karakter pertama menunjukkan jenis file (`-` untuk file biasa, `d` untuk direktori). Sembilan karakter berikutnya menunjukkan permission untuk tiga kelas: **User** (pemilik), **Group** (anggota grup), dan **Other** (pengguna lain). Setiap kelas memiliki tiga bit: `r` (read/4), `w` (write/2), dan `x` (execute/1).

Permission dapat diubah menggunakan perintah `chmod` dengan notasi oktal maupun simbolik. Perintah `umask` digunakan untuk mengatur izin default file yang baru dibuat.

### 15.2 Integritas (SHA-256 & Hashing)

Integritas data memastikan bahwa data tidak mengalami perubahan yang tidak sah. Hash berfungsi sebagai sidik jari digital dari sebuah file. Algoritma hash yang umum digunakan:
- **SHA-1**: menghasilkan hash 160-bit
- **SHA-256**: menggunakan blok 32-bit, menghasilkan hash 256-bit
- **SHA-512**: menggunakan blok 64-bit, menghasilkan hash 512-bit
- **MD5**: menghasilkan hash 128-bit

Sifat penting fungsi hash kriptografis adalah **avalanche effect**, di mana perubahan sekecil apapun pada input menghasilkan perubahan drastis pada nilai hash output. Perintah yang digunakan: `sha256sum`, `sha512sum`, `md5sum`. Tool GUI yang dapat digunakan adalah **GtkHash**.

### 15.3 Konfidensialitas

Konfidensialitas memastikan informasi hanya dapat diakses oleh pihak yang berwenang. Dua tool utama yang digunakan:

**EncFS** adalah tool untuk membuat filesystem terenkripsi di user space. Cara kerjanya menggunakan dua folder yang saling terkait: satu folder normal (plaintext) dan satu folder terenkripsi (ciphertext). Setelah di-unmount, folder normal menjadi kosong dan data hanya tersimpan dalam bentuk terenkripsi.

**GPG (GNU Privacy Guard)** merupakan implementasi sistem kriptografi kunci publik (PKI). Setiap pengguna memiliki pasangan kunci: kunci publik (dibagikan) dan kunci privat (disimpan rahasia). Pengirim menggunakan kunci publik penerima untuk mengenkripsi pesan, dan hanya penerima yang dapat mendekripsi menggunakan kunci privatnya.

```bash

sha256sum /home/praktikan/file_0.txt

Hal ini karena ketiga file memiliki konten yang identik. `test_0.txt` dibuat dengan menyalin isi `/etc/passwd`, dan `file_0.txt` hanya merupakan hasil rename dari `test_0.txt` sehingga kontennya tidak berubah sama sekali. Fungsi hash bersifat deterministik: input yang sama selalu menghasilkan output yang sama. Perubahan **nama file tidak mempengaruhi nilai hash** karena hash dihitung dari isi file, bukan dari namanya.

```

![alt text](image-5.png)

**c. Hapus titik di akhir file `test_1.txt`, lalu simpan.**

![alt text](image-6.png)

---

**d. Hash SHA256, SHA512, dan MD5 untuk `test_1.txt` (setelah diubah):**

```bash
sha256sum test_1.txt
sha512sum test_1.txt
md5sum test_1.txt
```

![alt text](image-7.png)

---

**e. Analisis:**

Nilai hash sebelum dan sesudah penghapusan titik **tidak sama**, meskipun hanya satu karakter yang dihapus. Ini adalah demonstrasi dari **avalanche effect** (efek longsoran), yaitu sifat fungsi hash kriptografis di mana perubahan kecil pada input menghasilkan perubahan yang sangat signifikan pada output hash. Fenomena ini membuktikan bahwa fungsi hash sangat sensitif terhadap setiap bit data, sehingga efektif untuk mendeteksi perubahan sekecil apapun pada sebuah file.

```

![alt text](image-9.png)

---

**c. Buka `test_1.doc`, ketik `abcdef`, simpan, hapus `abcdef`, simpan kembali.**

![alt text](image-10.png)

---

**d. Hash SHA256, SHA512, dan MD5 untuk `test_1.doc` setelah operasi edit:**

```bash
sha256sum test_1.doc
sha512sum test_1.doc
md5sum test_1.doc
```

![alt text](image-11.png)

---

**e. Hasil Pengamatan:**

Nilai hash file `test_1.doc` setelah operasi edit (ketik `abcdef` lalu hapus) **berbeda** dengan nilai hash awal, meskipun isi teks terlihat sama. Hal ini karena file `.doc` menyimpan **metadata** dokumen seperti tanggal modifikasi terakhir, riwayat revisi, dan informasi internal format file. Setiap kali file disimpan, metadata ini diperbarui sehingga mengubah byte-byte dalam file meskipun konten teksnya kembali sama. Ini menunjukkan bahwa hash mencerminkan **seluruh isi binary file**, termasuk metadata tersembunyi.

### 4. Konfidensialitas: EncFS [27 Point]

**a. Jalankan EncFS untuk membuat folder terenkripsi:**

```bash
encfs ~/Marsella_2311104004/folder_terenkripsi ~/Marsella_2311104004/folder_normal
```
*(Pilih y → y → Enter, buat password)*

![alt text](image-12.png)

---

**b. Copy file ke `folder_normal` dan amati `folder_terenkripsi`:**

```bash
cp /etc/passwd ~/Marsella_2311104004/folder_normal/
cp /etc/hostname ~/Marsella_2311104004/folder_normal/
ls ~/Marsella_2311104004/folder_terenkripsi/
```

![alt text](image-18.png)

**Observasi:** File yang disimpan di `folder_normal` tampil dengan nama dan isi yang dapat dibaca. Namun di `folder_terenkripsi`, file yang sama muncul dengan nama yang dienkripsi (nama acak) dan isinya tidak dapat dibaca karena terenkripsi. EncFS melakukan enkripsi transparan secara real-time.

---

**c. Hapus salah satu file di `folder_terenkripsi` dan amati `folder_normal`:**

```bash
ls ~/Marsella_2311104004/folder_terenkripsi/
rm ~/Marsella_2311104004/folder_terenkripsi/<nama_file_terenkripsi>
ls ~/Marsella_2311104004/folder_normal/
```

![alt text](image-19.png)

**Observasi:** Ketika satu file dihapus dari `folder_terenkripsi`, file yang bersesuaian juga hilang dari `folder_normal`. Ini karena kedua folder saling terkait — `folder_terenkripsi` adalah backend penyimpanan yang sebenarnya, sementara `folder_normal` adalah tampilan dekripsi real-time.

---

**d. Unmount `folder_normal` dan amati kedua folder:**

```bash
fusermount -u ~/Marsella_2311104004/folder_normal
ls ~/Marsella_2311104004/folder_normal/
ls ~/Marsella_2311104004/folder_terenkripsi/
```

![alt text](image-20.png)

**Observasi:** Setelah unmount, `folder_normal` menjadi kosong karena EncFS tidak lagi me-mount dekripsi. `folder_terenkripsi` tetap berisi file-file dengan nama terenkripsi. Data aman tersimpan dalam bentuk terenkripsi dan tidak dapat diakses tanpa mount ulang dengan password yang benar.

---

**e. Buat `folder_sembarang` dan mount ulang `folder_terenkripsi` ke sana:**

```bash
mkdir ~/Marsella_2311104004/folder_sembarang
encfs ~/Marsella_2311104004/folder_terenkripsi ~/Marsella_2311104004/folder_sembarang
ls ~/Marsella_2311104004/folder_sembarang/
```

![alt text](image-21.png)

**Observasi:** Setelah memasukkan password yang sama, `folder_sembarang` menampilkan isi yang sama seperti yang sebelumnya ada di `folder_normal`. Ini membuktikan bahwa `folder_terenkripsi` dapat di-mount ke direktori manapun dengan hasil yang sama selama menggunakan password yang benar.
