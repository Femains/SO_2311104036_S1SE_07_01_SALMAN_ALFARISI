## Dasar Teori

Linux adalah sistem operasi berbasis Unix yang menggunakan antarmuka berbasis teks (terminal/command line) sebagai salah satu cara utama berinteraksi dengan sistem. Perintah-perintah dasar Linux dieksekusi pada Bash (/bin/bash), yaitu sebuah shell yang menginterpretasikan perintah yang dimasukkan pengguna ke dalam konsol.

Struktur penulisan perintah Linux mengikuti format umum berikut:

```
$ Perintah [-opsi] [argument_1] [argument_2]
```

Penulisan perintah Linux bersifat **case sensitive**, artinya huruf besar dan huruf kecil dibedakan. Tanda `$` menunjukkan user biasa, sedangkan `#` menunjukkan user root yang memiliki hak akses penuh.

Beberapa perintah dasar yang dipelajari pada modul ini antara lain:

- **ls** – menampilkan isi direktori
- **cd** – berpindah direktori
- **cp** – menyalin file atau direktori
- **rm** – menghapus file atau direktori
- **mv** – memindahkan atau mengganti nama file
- **mkdir** – membuat direktori baru
- **pwd** – menampilkan direktori kerja saat ini
- **man** – menampilkan manual dari suatu perintah
- **| (pipe)** – menggabungkan output suatu perintah sebagai input perintah berikutnya
- **Redirection (>, >>)** – menyimpan output ke dalam file

Selain itu, pada modul ini juga dipelajari cara mengkompilasi program menggunakan **GCC (GNU Compiler Collection)**, yaitu compiler untuk bahasa C yang merupakan komponen kunci dari GNU Toolchain. Cara mengkompilasi source code menggunakan GCC adalah:

```
$ gcc source_code.c -o program_hasil
$ ./program_hasil
```

### 10. Redirection

#### a. Perintah `ls -al > /home/[username]/result.txt`

![alt text](image-21.png)

```bash
$ cd /
$ ls -al > /home/[username]/result.txt
```

**Penjelasan:** Perintah `ls -al` dijalankan dari direktori `/`, lalu seluruh outputnya tidak ditampilkan di terminal melainkan disimpan (ditulis) ke dalam file `result.txt` di folder home.

**b. Lokasi file result.txt:**
File `result.txt` berada di `/home/[username]/result.txt`

#### c. Perintah `ls -al > /home/[username]/result.txt` dari `/etc`

![alt text](image-22.png)

```bash
$ cd /etc
$ ls -al > /home/[username]/result.txt
```

**Penjelasan:** Perintah ini mengeksekusi `ls -al` dari direktori `/etc` dan menyimpan hasilnya ke `result.txt`. Karena menggunakan `>`, isi file `result.txt` sebelumnya (daftar direktori `/`) akan **ditimpa (overwrite)** dan digantikan dengan daftar isi direktori `/etc`.

**d. Fungsi perintah `>`:**
Perintah `>` adalah operator redirection yang menyimpan output dari suatu perintah ke dalam file. Jika file sudah ada, isinya akan **ditimpa (overwrite)**. Jika file belum ada, file baru akan dibuat.

#### e. Perintah `ls -al >> /home/[username]/result1.txt` dari `/`

![alt text](image-23.png)

```bash
$ cd /
$ ls -al >> /home/[username]/result1.txt
```

**Penjelasan:** Perintah ini menyimpan output `ls -al` dari direktori `/` ke dalam `result1.txt`. Jika file belum ada, file baru dibuat. Jika sudah ada, output ditambahkan di akhir file (append).

#### f. Perintah `ls -al >> /home/[username]/result1.txt` dari `/etc`

![alt text](image-24.png)

```bash
$ cd /etc
$ ls -al >> /home/[username]/result1.txt
```

**Penjelasan:** Perintah ini menambahkan (append) output `ls -al` dari direktori `/etc` ke akhir file `result1.txt`, sehingga file berisi dua bagian: daftar isi `/` diikuti daftar isi `/etc`.

**g. Perbedaan `>` dan `>>`:**
- `>` (overwrite): Menyimpan output ke file dan **menimpa** isi file jika sudah ada
- `>>` (append): Menyimpan output ke file dengan **menambahkan** di bagian akhir tanpa menghapus isi yang sudah ada

```c
    return 0;
}
```

#### 7. Penjelasan program `myopen`

Program `myopen` adalah program yang mencoba membuka file yang diberikan sebagai argumen menggunakan system call `open()` dengan mode read-only (`O_RDONLY`). Cara kerjanya:

1. Program memeriksa jumlah argumen – jika tidak tepat 2 (nama program + nama file), maka menampilkan petunjuk penggunaan dan keluar
2. Jika argumen benar, program mencoba membuka file yang disebutkan
3. Jika berhasil dibuka, program mencetak pesan sukses
4. Jika gagal (misalnya file tidak ditemukan atau tidak punya izin), program mencetak pesan error beserta keterangan error dari sistem (`strerror(errno)`)
